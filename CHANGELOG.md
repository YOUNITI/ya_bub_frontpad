## [2026-04-21] - Исправлена ошибка добавления допов к размерам товара

### Проблема
При попытке добавления допов к размерам товара возникала ошибка:
```
Error adding size-addon: Cannot add or update a child row: a foreign key constraint fails
```

### Причина
1. **Ошибка 1 (фронтенд)** - В функции `handleAddAddonToSize` переменная `updatedSizeAddons` обновлялась, но в стейт записывался старый объект `newSizeAddons`
2. **Ошибка 2 (фронтенд)** - При замене временных ID размеров на реальные ID из базы данных сравнивался строковый ключ с числовым ключом, поэтому замена не происходила
3. **Ошибка 3 (бэкенд)** - В endpoint `/api/products/:productId/sizes/:sizeId/addons` INSERT запрос использовал несуществующие столбцы `addon_name` и `addon_price` вместо `addon_id`, `price_modifier`, `is_required`
4. **Ошибка 4 (бэкенд)** - В endpoint `/api/sizes/:sizeId/addons` не было проверки существования размера перед добавлением допа
5. **Ошибка 5 (конфигурация)** - LOCAL_API_BASE был указан как порт 3001, но сервер работает на порту 3005

### Решение
1. ✅ Исправлена переменная в `handleAddAddonToSize` - теперь обновляется правильный объект
2. ✅ Исправлена замена ID в sizeAddons - теперь ключ преобразуется в число перед поиском
3. ✅ Удаляются старые временные ID из sizeAddons после замены
4. ✅ Добавлена проверка существования размера в endpoint `/api/sizes/:sizeId/addons`
5. ✅ Исправлен INSERT запрос в бэкенде - теперь используются корректные столбцы таблицы `size_addons`
6. ✅ Обновлен LOCAL_API_BASE с 3001 на 3005
7. ✅ Размеры теперь сохраняются всегда, даже при редактировании существующего товара

### Файлы изменены:
- `var/www/ya_budu/ya_budu/frontpad/client/src/components/Products.jsx`
- `var/www/ya_budu/ya_budu/frontpad/server/src/frontpad-server.js`

## [2026-04-20] - Исправлена ошибка сохранения дополнительных ингредиентов для размеров

### Проблема
При сохранении дополнительных ингредиентов для размеров товаров возникала ошибка Foreign key constraint fails и допы не сохранялись.

### Причина
1. При сохранении товара удалялись ВСЕ старые размеры и создавались новые
2. Но при этом использовались временные ID (Date.now()) которые не существовали в базе данных
3. При попытке сохранить допы для этих временных ID возникала ошибка внешнего ключа

### Решение
1. ✅ Теперь после создания новых размеров мы получаем и сохраняем их реальные ID из базы данных
2. ✅ Обновляем массив `sizes` и `sizeAddons` этими реальными ID
3. ✅ Исправлена отправка поля `price_modifier` вместо `price` при создании размера (из-за этого цена размера всегда была 0)

### Файлы изменены:
- `var/www/ya_budu/ya_budu/frontpad/client/src/components/Products.jsx` - исправлена логика сохранения размеров и сопоставление ID для size-addons

   - Визуально отображается, когда доставка будет бесплатной, а когда платной
   - Динамическое обновление превью при изменении полей формы

3. **Улучшено отображение таблицы:**
   - Добавлена информация о ID района

- Добавлены стили для модального окна увеличенного размера

### Функциональность

**Полный цикл управления районами доставки:**

1. **Создание района:**
   - Администратор заходит в Frontpad → Районы доставки → Добавить район
   - Заполняет поля: название, минимальная сумма, стоимость доставки, порядок
   - Видит превью расчета для разных сумм заказа
   - Сохраняет район

2. **Редактирование района:**
   - В таблице районов нажимает кнопку редактирования
   - Изменяет параметры района
   - Видит обновленное превью расчета
   - Сохраняет изменения

3. **Удаление района:**
   - В таблице нажимает кнопку удаления
   - Подтверждает удаление

4. **Активация/деактивация:**
   - Кнопка в таблице для быстрого переключения статуса
   - Неактивные районы не показываются на основном сайте

5. **Синхронизация с основным сайтом:**
   - Районы автоматически доступны на основном сайте через `/api/site/delivery-zones`
   - При оформлении заказа клиент выбирает район
   - Система автоматически рассчитывает стоимость доставки
   - Вся информация передается в Frontpad при синхронизации заказа

### API Endpoints (уже реализованы)

- `GET /api/delivery-zones` - получить все активные районы
- `GET /api/delivery-zones/:id` - получить район по ID
- `POST /api/delivery-zones` - создать район
- `PUT /api/delivery-zones/:id` - обновить район
- `DELETE /api/delivery-zones/:id` - удалить район
- `GET /api/site/delivery-zones` - получить районы для основного сайта (с токеном)
