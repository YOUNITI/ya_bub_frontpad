### 1. **Авторизация пользователей**
- ✅ `admin` (точка 1) получает токен с `point_id: 1`
- ✅ `admin2` (точка 2) получает токен с `point_id: 2`
- ✅ WebSocket авторизация правильно сохраняет `pointId` клиента

### 2. **Синхронизация заказов с основного сайта**
- ✅ При создании заказа определяется `point_id` района доставки
- ✅ `point_id` передается в Frontpad при синхронизации
- ✅ Заказ сохраняется с правильным `point_id`

### 3. **WebSocket broadcast фильтрация**
```javascript
// ПРАВИЛЬНАЯ логика:
if (client.pointId === 0) {
  // Главный админ получает ВСЕ заказы
  client.send(message);
  return;
}

if (client.role === 'admin' && client.pointId && data.order.point_id === client.pointId) {
  // Админ точки получает ТОЛЬКО заказы своей точки
  client.send(message);
  return;
}

if (client.role === 'courier' && client.pointId && data.order.point_id === client.pointId) {
  // Курьер получает ТОЛЬКО заказы своей точки
  client.send(message);
  return;
}
```

- ✅ `/api/delivery-zones` показывает только районы своей точки
- ✅ `/api/reports/*` фильтрует по точке

### 5. **Фронтенд**
- ✅ Районы доставки фильтруются по точке
- ✅ Заказы отображаются только своей точки

```sql
-- Проверяем распределение заказов
SELECT p.name, COUNT(o.id) as orders_count
FROM points p
LEFT JOIN orders o ON p.id = o.point_id
WHERE p.is_active = 1
GROUP BY p.id, p.name;

-- Проверяем районы по точкам
SELECT p.name, COUNT(dz.id) as zones_count
FROM points p
LEFT JOIN delivery_zones dz ON p.id = dz.point_id AND dz.is_active = 1
WHERE p.is_active = 1
GROUP BY p.id, p.name;
```
