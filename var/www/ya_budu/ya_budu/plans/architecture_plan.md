# Архитектурный план SPA "Я Буду!" (Food Delivery App)

## 1. Общая архитектура приложения

### Стек технологий:
- React.js (Functional Components + Hooks)
- Tailwind CSS (для стилизации)
- React Context API или Redux Toolkit (для управления состоянием)
- React Router (хотя приложение SPA без роутинга, может понадобиться для будущего расширения)
- Axios/Fetch (для API вызовов, если потребуется)

### Тип архитектуры:
Одностраничное приложение (SPA) с вертикальной прокруткой, содержащее все необходимые секции на одной странице.

## 2. Компонентная структура

### Корневой компонент:
- `App.js` - основной компонент, объединяющий все остальные

### Основные компоненты:
- `Header` - шапка приложения с навигацией и информацией о пользователе
- `Hero` - приветственная секция с призывом к действию
- `MenuSection` - раздел меню с категориями и продуктами
- `CategoryList` - список категорий блюд
- `ProductCard` - карточка отдельного продукта
- `CartPanel` - панель корзины с товарами и стоимостью
- `Footer` - нижняя часть сайта с контактной информацией

## 3. Иерархия компонентов

```

│       └── ProductCard (множество экземпляров)
└── Footer
└── CartPanel (плавающая/модальная панель)
```

## 4. Управление состоянием

### Глобальное состояние (React Context или Redux):
- `cartState` - содержит товары в корзине, количество, общую стоимость
- `selectedCategory` - текущая выбранная категория меню
- `language` - выбранный язык интерфейса (русский/английский)
- `userLocation` - местоположение пользователя для доставки
- `deliveryTime` - выбранное время доставки

### Локальное состояние компонентов:
- `Header`: мобильное меню открыто/закрыто
- `ProductCard`: hover состояние, кнопка добавления
- `CartPanel`: открыта/закрыта, состояние оформления заказа
- `CategoryList`: активная категория

## 6. Подробное описание компонентов

### Header Component
- Логотип "Я Буду!"
- Навигационное меню (в данном случае ссылки на секции на той же странице)
- Информация о пользователе (иконка профиля, имя)
- Языковой переключатель (RU/EN)
- Иконка корзины с количеством товаров

### Hero Section
- Приветственный заголовок
- Подзаголовок с описанием сервиса
- Кнопка "Заказать сейчас"
- Фоновое изображение с едой
- Информация о доставке (бесплатно, быстро и т.д.)

### MenuSection
- Заголовок секции "Меню"
- CategoryList с горизонтальной прокруткой
- ProductGrid для отображения продуктов в выбранной категории
- Возможность фильтрации по популярности/цене

### CategoryList Component
- Горизонтальный скролл с категориями (пицца, суши, бургеры и т.д.)
- Выделение активной категории
- Отображение количества товаров в категории

### ProductCard Component
- Изображение продукта
- Название и описание
- Цена
- Кнопка "Добавить в корзину"
- Количество (для уже добавленных товаров)

### CartPanel Component
- Плавающая/модальная панель с товарами
- Список товаров с изображениями, названиями и ценами
- Возможность изменения количества или удаления
- Общая сумма заказа
- Кнопка "Оформить заказ"

### Footer Component
- Контактная информация
- Социальные сети
- Политика конфиденциальности
- Копирайт

## 9. Управление состоянием корзины

Корзина будет управляться через глобальный контекст (CartContext), который предоставляет следующие функции:
- addToCart(productId, quantity)
- removeFromCart(productId)
- updateQuantity(productId, quantity)
- clearCart()
- calculateTotal()

Это позволяет любому компоненту (ProductCard, CartPanel) взаимодействовать с корзиной без передачи пропсов на несколько уровней вложенности.

## 10. Переключение языка

Язык интерфейса будет управляться через LanguageContext, позволяя переключаться между русским и английским языками. Все строки будут храниться в виде объектов перевода.
